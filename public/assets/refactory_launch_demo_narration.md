# Refactory Launch Demo Narration

## Voice Direction

Calm, confident, product-demo tone. Keep it clear and direct. Moderate pace, with short pauses between sections.

## Full Script

Refactory is a workflow automation surface for the Superprism stack.

It helps communities turn chat, meetings, docs, and memory into repeatable operating workflows.

Start with a plain-language request.

Create a new workflow to create a Daily Brief from community memory, run daily at 9am, and send notification to Discord.

Refactory turns that intent into a durable workflow.

Requests capture what someone wants done. Workflows encode the process. Tasks and agents move the work forward. Artifacts preserve the outputs.

That means the work does not disappear into a chat thread.

Operators can inspect state, review steps, retry execution, and see what happened across the system.

The workflow can also pull from Prism Memory.

Memory keeps the record of what happened across meetings, Discord activity, summaries, and generated artifacts.

Knowledge gives agents stable reference material.

Together, they give the workflow enough context to produce useful briefs without starting from scratch every time.

Once the workflow is running, Refactory makes progress visible.

The system can run on a schedule, move through review and render steps, store the final output, and notify the community when the work is ready.

This is the core idea behind Refactory.

Community memory, agent execution, and operator workflows, connected in one self-hostable stack.

Refactory for Superprism.

Build the workflow.

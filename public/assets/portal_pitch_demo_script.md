Demo Goal
  Show Portal as the emerging coordination layer for RaidGuild: a place to see
  what is happening, find people/projects/sessions, and let agents help keep it
  updated.

  Opening: 1 min
  “Today I want to show the RaidGuild Portal MVP.

  The goal is not to replace Discord, Notion, GitHub, or any existing workflow.
  The goal is to give RaidGuild a public and member-facing coordination layer: a
  place where people can understand what is happening, find useful next steps,
  and connect the dots between members, sessions, projects, posts, and agents.

  This is still early. The main CTA today is: join the Portal, claim or create
  your member profile, and help us iterate.”

  Home / Brief: 2 min
  Start at https://portal.raidguild.org/.

  “First, the public homepage is meant to answer: what is RaidGuild doing right
  now?

  We have the current brief, sessions, projects, posts, and featured spotlights.
  The idea is that someone new should not have to lurk in Discord for weeks to
  understand where energy is moving.

  This becomes especially useful for cohort seasons, fireside chats, project
  spikes, and active campaigns.”

  Point out:

  - Current brief
  - Spotlight / featured thread if available
  - Sessions
  - Public posts/projects

  Suggested line:
  “The Portal is a presentation layer for real activity, not a generic marketing
  site.”

  Join / Onboarding: 2 min
  Click join/login path.

  “The onboarding flow is intentionally lightweight. We want people to create an
  account, then build or claim their profile.

  For existing RaidGuild members, the important action is claiming your old
  member profile. We imported legacy member profile data, so instead of everyone
  starting from scratch, people can connect their account to their existing
  identity.”

  Point out:

  - Join CTA
  - Profile claim/create profile
  - Profile roles/skills/links
  - Public member profile

  Suggested line:
  “Profiles are not just user accounts. They are how the community finds people
  by skills, roles, work, sessions, and contributions.”

  Member Dashboard: 2 min
  Log in and show dashboard.

  “Once logged in, the member home is a lightweight command center. You can see
  the current brief, sessions, projects, points/check-ins, modules, and your own
  profile state.”

  Point out:

  - Welcome message
  - Guild points / vibe check
  - Inbox if relevant
  - Modules
  - Public profile link
  - Sessions/projects/posts links

  Suggested line:
  “This is where we can start turning passive community awareness into active
  contribution paths.”

  Sessions: 3 min
  Go to Sessions.

  “Sessions are becoming the main MVP surface. RaidGuild has a lot of knowledge
  in live conversations, Discord calls, firesides, workshops, and demos. The
  Portal gives those sessions a durable page.”

  Show:

  - Upcoming sessions
  - Past sessions
  - Session detail page
  - Join / calendar / Discord links
  - Resources
  - YouTube embed if available
  - Comments
  - Related projects/threads/posts
  - Edit link for hosts/admins

  Suggested line:
  “A session can start as a calendar event, then become an archive: notes,
  recording, transcript, summary, resources, related projects, and eventually
  derived posts or wiki material.”

  Agent / Discord Bot Integration: 3 min
  This is likely the most interesting part.

  “Where this gets exciting is that Portal is not only manually edited through
  the CMS. Agents can help keep it current.

  For example, from Discord chat or a bot workflow, an agent can create or
  update a Portal session. It can attach source material after a meeting. It can
  link a Discord scheduled event, add resources, and update related fields.”

  Demo or describe:

  - “Create a session from chat”
  - “Attach YouTube/summary/transcript as resources”
  - “Update CMS copy for onboarding pages from chat”
  - “Create drafts or proposals instead of silently publishing”

  Suggested phrasing:
  “The important pattern is reviewable agent administration. Agents can do
  useful CMS work, but they should preserve source context and avoid inventing
  activity. This is not ‘AI content spam.’ It is operational help for keeping
  the community memory current.”

  If you have a bot prompt ready:
  “Queen Raida, create a Portal session for [event], link this Discord event,
  make it public, and add this summary.”

  Then:
  “Behind the scenes, the agent uses the Portal skill instructions so it knows
  the right API paths, what should be drafted, what can be published, and how to
  avoid bad writes.”

  CMS Copy From Chat: 1-2 min
  Show onboarding/inquiry page or explain.

  “Another useful pattern is CMS-managed page copy. If we want to update
  onboarding language, intake pages, or campaign copy, we do not need a code
  deploy every time.

  An agent can help propose or update the copy in Payload, and BD/community
  contributors can review the wording.”

  Suggested line:
  “This lets the people closest to the message shape the page without needing to
  become Next.js developers.”

  MVP / Contribution Ask: 2 min
  End with the CTA.

  “This is MVP-quality. It is intentionally not finished.

  What we need now is community usage and feedback:

  - Join the Portal.
  - Claim or create your member profile.
  - Check if your skills, roles, links, and public profile look right.
  - Join sessions and add resources.
  - Suggest projects, threads, and contribution requests.
  - Help improve the agent workflows.
  - Help polish onboarding, copy, and member experience.”

  Close:
  “The Portal should become RaidGuild’s living coordination layer: public enough
  for newcomers, useful enough for members, and structured enough for agents to
  help keep it fresh.

  The CTA is simple: go to portal.raidguild.org, join, and claim your profile.”

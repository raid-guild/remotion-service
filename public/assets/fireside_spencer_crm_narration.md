# Fireside Recap Narration: Spencer / Personal CRM

## Intro
Fireside recap: Spencer on private relationship memory, agentic coding loops, and what gets harder when every builder has stronger tools.

## Private relationship memory
Spencer is building a self-hosted personal CRM to keep relationship context private while helping him remember birthdays, life events, follow-ups, and important relationship history.

## Doer reviewer loop
His agentic coding workflow uses a doer and reviewer loop. Claude Code coordinates the work, Codex-style reviewers check plans and implementations, and deterministic checks backstop the process.

## Context and verification
Context and verification were the hard problems. The session kept returning to two questions: how do agents know enough context, and how do humans know the agent did good work?

## UX QA bottleneck
UX QA is becoming a major bottleneck for AI-built products. Spencer is exploring agents that inspect human-facing interfaces, open issues, and eventually help fix them.

## Atomized developer
The biggest coordination concern was the atomized developer problem. Agents may make individuals more capable, but weaken the shared context, relationships, and collaboration that make project work meaningful.
